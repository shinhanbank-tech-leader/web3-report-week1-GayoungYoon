# 블록체인의 신뢰 형성 구조와 기존 금융 시스템의 신뢰 구조 비교
## - RDBMS 시스템과의 비교를 중심으로
### 1. 신뢰 형성 구조의 개요

기존 금융 시스템과 블록체인은 모두 거래를 안전하게 기록하고 자산 이동의 정당성을 보장해야 한다는 공통 목적을 가진다. 그러나 두 시스템이 신뢰를 형성하는 방식은 크게 다르다. 기존 금융 시스템은 은행, 결제기관, 중앙은행과 같은 중앙 기관을 중심으로 신뢰를 구축하며, 거래 기록과 잔액 정보는 주로 중앙 데이터베이스(RDBMS)에 저장된다. 이때 정합성은 트랜잭션, lock, isolation level 등의 데이터베이스 관리 기법을 통해 유지된다.

반면 블록체인은 특정 기관을 신뢰의 중심에 두지 않는다. 대신 네트워크 참여자들이 동일한 규칙을 따르도록 하고, 암호학적 서명과 합의 메커니즘을 통해 거래의 유효성과 기록의 정당성을 검증한다. 즉 기존 금융 시스템이 기관에 대한 신뢰를 바탕으로 작동한다면, 블록체인은 규칙과 알고리즘에 대한 신뢰를 바탕으로 작동한다고 볼 수 있다.

### 2. 정합성 관리 방식의 차이

기존 금융 시스템에서 정합성은 중앙 DB가 즉시 관리한다. 예를 들어 계좌 이체는 하나의 트랜잭션으로 처리되며, 송금과 입금이 함께 반영되거나 함께 취소된다. 이 과정에서 lock이 적절히 설정되지 않으면 여러 사용자가 동시에 같은 데이터를 수정할 때 lost update와 같은 문제가 발생할 수 있다. 따라서 RDBMS는 ACID, row lock, MVCC 등을 통해 강한 일관성을 유지한다.

반면 블록체인에서는 거래가 발생하더라도 곧바로 확정되지 않는다. 먼저 네트워크에 전파되어 각 노드의 mempool에 저장되고, 이후 채굴자나 검증자가 이를 블록에 포함시키면 확정된다. 여기서 중요한 점은 mempool이 전체 네트워크가 공유하는 하나의 공간이 아니라 각 노드가 개별적으로 가진 로컬 대기열이라는 점이다. 따라서 같은 시점에도 노드마다 본 거래가 다를 수 있다. 블록체인은 이러한 차이를 합의 메커니즘을 통해 최종적으로 하나의 체인으로 수렴시킨다. 즉 RDBMS가 즉시 정합성을 추구한다면, 블록체인은 최종 정합성(eventual consistency) 에 가깝다.

### 3. 신뢰 형성 방식의 핵심 차이

기존 금융 시스템의 신뢰는 중앙 기관의 책임성과 통제력에서 나온다. 사용자는 자신의 잔액이나 거래 기록을 직접 검증하지 않고, 은행 시스템이 이를 정확히 관리한다고 믿는다. 문제가 발생했을 때도 관리자 권한, 감사 체계, 법적 책임 등을 통해 복구와 수정이 가능하다.

반면 블록체인은 중앙 관리자가 없기 때문에, 거래 당사자가 개인키로 서명한 내용이 맞는지 공개키로 검증하고, 블록이 네트워크 규칙에 맞는지 모든 노드가 independently 확인한다. 그리고 최종적으로는 가장 많은 작업량이 쌓인 체인 또는 합의 규칙상 정당한 체인을 채택한다. 즉 블록체인의 신뢰는 특정 주체의 선의가 아니라 암호학과 합의 규칙에 의해 형성되는 구조적 신뢰라고 할 수 있다.

### 4. 각 시스템의 취약점

기존 금융 시스템은 빠르고 효율적이지만 중앙화 구조에 따른 취약점이 존재한다. DB 서버 장애가 발생하면 전체 서비스가 멈출 수 있고, 관리자 실수나 내부자 공격으로 데이터가 잘못 수정될 가능성도 있다. 또한 동시성 제어가 적절하지 않으면 lock 충돌이나 race condition으로 인해 정합성이 깨질 수 있다.

블록체인은 중앙 기관 없이도 신뢰를 유지할 수 있다는 장점이 있지만, 다른 유형의 위험을 가진다. 거래가 블록에 포함되기 전에는 0-confirmation 상태이므로 double spend 가능성이 완전히 사라지지 않으며, 동시에 두 블록이 생성되면 일시적인 fork가 발생할 수 있다. 또한 작업증명 기반 시스템에서는 51% 공격과 같은 합의 구조 자체에 대한 위협도 존재한다. 즉 RDBMS는 중앙 통제 실패가 위험이고, 블록체인은 합의 실패 또는 체인 분기가 위험이라고 정리할 수 있다.

### 5. 종합 비교

두 시스템의 차이는 “무엇을 신뢰의 기반으로 삼는가”에서 가장 분명하게 드러난다. 기존 금융 시스템은 중앙 기관이 데이터를 통제하고 보증하는 구조이며, RDBMS를 통해 빠르고 강한 일관성을 제공한다. 반면 블록체인은 데이터를 여러 참여자가 나누어 보관하고, 합의 메커니즘을 통해 동일한 상태로 수렴한다. 따라서 기존 금융 시스템은 효율성과 관리 용이성이 강점이고, 블록체인은 탈중앙성, 검열 저항성, 불변성이 강점이다.

### 6. 결론

기존 금융 시스템과 블록체인은 모두 거래의 신뢰를 확보하려는 목적을 가지지만, 신뢰를 만들어내는 방식은 근본적으로 다르다. 기존 금융 시스템은 중앙 기관과 RDBMS의 통제를 통해 정합성을 유지하고, 블록체인은 암호학과 합의 메커니즘을 통해 참여자 간 공통된 원장을 형성한다. 정리하면 기존 금융 시스템은 기관 중심의 신뢰, 블록체인은 규칙 중심의 신뢰라고 볼 수 있다. 따라서 두 구조는 단순히 어느 한쪽이 더 우수하다고 보기보다, 요구되는 신뢰의 방식과 시스템 목적에 따라 서로 다른 장점을 가진 구조로 이해하는 것이 적절하다.
